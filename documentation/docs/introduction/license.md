# License

ESDM is **open source**. Everything that makes up ESDM – the source code of the `esdm` binary, the schema that defines what a valid model looks like, and this documentation – is released under the **[MIT license](https://opensource.org/license/mit)**. The full license text ships with every copy of the **[repository](https://github.com/thenativeweb/esdm)** as `LICENSE.md`.

The authoritative version of the schema lives at **[https://www.esdm.io](https://www.esdm.io)**. If a definition seems wrong or missing, we'd rather hear about it than discover an incompatible variant in the wild – write to us at **[hello@thenativeweb.io](mailto:hello@thenativeweb.io)**.

## How ESDM Is Maintained

ESDM is developed and maintained by **[the native web GmbH](https://www.thenativeweb.io)**, a Germany-based consultancy specializing in **Event Sourcing**, **CQRS**, **Domain-Driven Design**, and **event-driven architectures**. ESDM grew out of the modeling work we do with our clients – the vocabulary you find in the schema is the same vocabulary we use every day.

## Reporting Issues

If you run into a bug, a misleading diagnostic, a rule that flags something it shouldn't, or a feature you'd like to see, open an issue in the **[ESDM repository on GitHub](https://github.com/thenativeweb/esdm/issues)**. For commercial questions, integration support, or a general conversation about Event Sourcing and Domain-Driven Design, write to **[hello@thenativeweb.io](mailto:hello@thenativeweb.io)**.
