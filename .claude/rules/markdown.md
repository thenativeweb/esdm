# Markdown

- Separate any two block elements with exactly one blank line (heading and heading, heading and paragraph, paragraph and paragraph, and so on). A fenced code block (listing) counts as a block element too: leave exactly one blank line before it and one after it.
- Use asterisks for emphasis: `*italic*` for italic, `**bold**` for bold.
- Use a hyphen (`-`) as the list marker.
- Always annotate fenced code blocks with a language tag.
  - For shell commands, use `shell`, not `bash`, unless the snippet is specifically about Bash.
- Do not hard-wrap text at a fixed column width; write each paragraph as a single line.
- Use straight quotation marks (`"`) for quotations; do not use typographic (curly) quotation marks.
